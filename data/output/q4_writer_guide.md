# Q4 当前实现说明（最终优化版）

## 1. 题目任务对齐

第四问要求在机器人存在故障风险的条件下，结合人工复位最多 2 次、战术暂停最多 2 次、紧急故障维修最多 1 次等资源约束，以单败淘汰 BO3 三局两胜的最终获胜概率最大化为目标，给出领先、平局、落后等不同比赛场景下资源的最优使用时机与次数。

当前 Q4 的实现口径是：

> 在最终版 Q3 单局策略模型基础上，构建“局内有限时域 MDP + BO3 宏观动态规划”的双层资源调度模型。局内层决定战术动作和资源动作，宏观层决定每一局分配多少资源，并按实际使用量扣减跨局库存。

## 1.1 一个优秀 Q4 答案应满足什么

优秀的第四问答案不应只给一个 BO3 胜率，而应同时回答四件事：

1. **接口一致**：无资源、无额外资源收益时，Q4 应复现最终版 Q3 的领先、平局、落后三场景基础胜率；
2. **资源有边界**：资源能提高平局局和落后局的应对能力，但不能把落后局从极低胜率直接推成高胜率；
3. **时机可解释**：必须说明什么时候复位、什么时候暂停、什么时候维修，以及领先时为什么保留资源；
4. **BO3 动态性**：主方法应体现跨局资源保留与再分配价值，并优于静态配额、首局全投和固定规则。

按这个标准审查，当前 Q4 方向是正确的：它继承 Q3 基础胜率和动作核，显式加入故障、倒地、暂停、复位、维修和 BO3 状态，并输出资源时机规则。当前主要边界是：Q4 仍使用宏观战术动作核，而不是完整 35 个原子动作。

## 2. 当前建模链路

运行入口：

```bash
python scripts/run_q4.py
```

主流程如下：

1. 读取 Q1 动作特征、Q2 攻防 pair 表、最终版 Q3 动作核和 Q3 方法胜率；
2. 读取 Q4 故障与资源参数；
3. 从 Q3 动作核压缩得到 Q4 的宏观战术动作；
4. 对 18 种单局资源配额 `(reset, pause, repair)` 分别求局内 MDP；
5. 用最终版 Q3 三场景胜率校准 Q4 零资源基线；
6. 生成单局资源配额胜率表 `P_win` 和实际资源使用分布；
7. 在 BO3 层做宏观动态规划；
8. 对最优 DP、穷举静态、首局全投、固定规则四种方法做 Monte Carlo 对比；
9. 输出诊断表和图表。

## 4. 局内 MDP 状态与动作

局内状态变量为：

- `score_diff`：当前分差；
- `time_bucket`：局内时间桶；
- `health_my / health_opp`：双方机能档；
- `fault`：是否处于故障状态；
- `down_flag`：是否倒地或失稳；
- `reset_left / pause_left / repair_left`：本局剩余资源。

局内动作分两类。

常规战术动作：

- 试探攻击；
- 激进攻击；
- 反击防守；
- 保守防守；
- 平衡恢复。

资源与等待动作：

- 人工复位；
- 战术暂停；
- 紧急维修；
- 倒地等待；
- 故障等待。

本轮优化后，Q4 的宏观战术核不再只取每类 Top1 动作，而是对 Q3 同一宏观类下的 Top-k 动作做加权聚合，降低单一动作替身导致的偏差。

## 5. 故障与资源动作修正

故障率当前采用“机能 + 时间累积 + 动作负荷”的轻量模型：

```text
fault_rate
= lambda_0 * exp(k_fault * (1 - health_ratio))
  * (1 + fault_time_weight * time_progress)
  * (1 + fault_action_weight * action_load)
```

其中动作负荷按宏观动作区分：

- 激进攻击负荷较高；
- 反击防守中等；
- 试探攻击和保守防守较低；
- 平衡恢复有负向修正。

资源动作也被修正为“有代价的收益动作”：

- 战术暂停：恢复机能，但消耗机会桶，并有小概率失分；
- 人工复位：清除倒地/失稳，但有复位期间失分风险；
- 紧急维修：清除故障并恢复机能，但机会桶代价最高，且维修期间可能丢分。

## 6. BO3 宏观动态规划

宏观状态为：

```text
(wins_my, wins_opp, reset_left, pause_left, repair_left)
```

每个状态下，模型根据当前 BO3 比分判断下一局场景：

- `wins_my > wins_opp`：领先局；
- `wins_my = wins_opp`：平局局；
- `wins_my < wins_opp`：落后局。

宏观层选择本局配额 `(reset_alloc, pause_alloc, repair_alloc)`，目标是最大化 BO3 最终获胜概率。

当前最重要的正确性改进是：

> 资源不是按“分配量”直接扣减，而是按局内 MDP 推出的“实际使用量分布”扣减。

这避免了“本局给了 2 次复位就必然消耗 2 次”的错误假设。

## 7. 当前核心结果

### 7.1 单局资源增益

当前最优配额下的单局胜率为：

| 场景 | 零资源胜率 | 最优资源胜率 | 最优配额 |
|---|---:|---:|---|
| 领先局 | 1.000 | 1.000 | R0-P0-M0 |
| 平局局 | 0.544 | 0.617 | R2-P2-M1 |
| 落后局 | 0.040 | 0.177 | R2-P2-M1 |

解释：

- 领先局由于 Q3 基础胜率已达到 1.000，宏观层选择保留资源；
- 平局局资源带来中等提升，是最主要的资源投入场景；
- 落后局资源能提高翻盘概率，但不会被不合理地拉高到 0.8 以上。

### 7.2 BO3 方法对比

Monte Carlo 5000 组 BO3 仿真结果：

| 方法 | BO3 胜率 | 95% CI |
|---|---:|---|
| 最优 DP | 0.658 | [0.645, 0.671] |
| 穷举静态 | 0.620 | [0.606, 0.633] |
| 首局全投 | 0.616 | [0.603, 0.629] |
| 固定规则 | 0.454 | [0.441, 0.468] |

结论口径：

> 最优 DP 相比静态分配和首局全投有稳定优势，优势主要来自“按 BO3 比分动态调整资源配额”和“按实际使用量保留未触发资源”。

## 8. 当前推荐资源规则

当前 `q4_resource_timing_rules.csv` 给出的核心规则是：

1. 领先局：不主动投入资源，优先保留给后续平局或落后局；
2. 平局局：推荐满配额 `R2-P2-M1`，暂停最常用，通常较早触发；维修主要在中后段故障状态下触发；复位由倒地/失稳状态触发；
3. 落后局：同样推荐 `R2-P2-M1`，但资源只能提高翻盘概率，不能保证反转；暂停和维修都属于选择性使用；
4. 故障状态：若维修剩余且时间不是极端不足，紧急维修优于长期故障等待；
5. 倒地/失稳状态：若复位剩余，人工复位通常优于等待；
6. 低机能状态：战术暂停主要用于降低后续故障和失分风险。

对应文件：

- `data/interim/q4_resource_timing_rules.csv`
- `data/interim/q4_resource_trigger_states.csv`
- `data/interim/q4_resource_value_gap.csv`
- `data/interim/q4_resource_value_gap_summary.csv`

其中 `q4_resource_value_gap.csv` 用于解释“为什么此时用资源”：它对资源触发相关状态重新计算所有可行动作的 Q 值，输出 `top1_action`、`top2_action`、`best_resource_action`、`resource_gain_vs_non_resource` 和 `resource_gain_vs_wait`。这张表是资源时机规则的价值差证据。

新增 `q4_resource_policy_heatmap_table.csv` 和 `q4_resource_policy_heatmap.png`，用于展示不同分差、时间段和状态条件下的主推荐动作。它直接服务第四问中的“最优使用时机”。

## 10. 论文表述边界

不要写：

> Q4 求得了真实比赛中的绝对最优资源方案。

建议写：

> 本文在最终版 Q3 单局策略模型基础上，构建了局内有限时域 MDP 与 BO3 宏观动态规划相结合的资源调度模型。模型显式区分故障维修、倒地复位和战术暂停，并通过最终 Q3 三场景胜率对 Q4 零资源基线进行校准，从而保证资源增益建立在一致的单局策略口径上。结果表明，动态资源调度在保持领先局资源保留的同时，能显著提高平局局和落后局的应对能力，并在 BO3 总体胜率上优于静态分配、首局全投和固定规则。

当前仍需说明的边界：

- Q4 为控制状态空间，使用 Q3 的宏观战术动作核，而非完整 35 个原子动作；
- 故障率函数是基于机能、时间和动作负荷的可解释近似模型；
- 资源动作的时间代价采用“战术机会桶”口径，而不是逐秒复刻比赛暂停时间；
- 领先局基础胜率已为 1.000，因此资源增益自然为 0，不能再强调领先局资源提升。
- 题面背景提到备用机器人，但第四问主体要求明确围绕人工复位、战术暂停和紧急故障维修三类资源展开。当前模型将备用机器人视为不可逆故障后的赛制边界条件，没有纳入局内 MDP 主体。
